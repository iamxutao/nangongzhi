## 一、角色定义 (Actors)

| 角色 | 描述 | 核心关注点 |
|------|------|------------|
| **设备管理员** | 负责设备台账、结构配置 | 设备建模、数据点配置 |
| **工艺工程师** | 负责规则配置、阈值调整 | 检测规则、报警策略 |
| **维修工程师** | 负责故障处理、维修执行 | 故障诊断、维修指导 |
| **车间主管** | 负责生产调度、资源协调 | 设备健康总览、报表分析 |
| **知识专家** | 负责专家库维护 | 诊断树、知识沉淀 |

## 三、用户故事 (User Stories)

### 🔧 领域1：设备建模域 (Equipment Context)

#### US-EQ-001 配置设备结构
```
作为 设备管理员，
我希望 能够从IoT现有设备管理中选择机床设备，配置其层级结构（整机→子系统→部件），
以便 建立完整的设备资产模型，作为健康评估的对象基础。

验收标准：
【新增配置】
- 从IoT设备管理中选择机床设备（无需新增设备）
- 支持三级层级结构：整机、子系统、部件
- 每个节点可配置权重、关键等级
- 支持按机床型号维护设备结构模板
- 配置时自动匹配同型号模板，微调后一键生成

【修改配置】
- 支持对已配置的设备结构进行节点级修改（增删改）
- 删除节点时级联删除所有子节点
- 根节点（整机）不允许删除

【删除配置】
- 支持删除整个设备结构（移除该设备的全部结构配置）
- 删除后，设备可重新选择模板进行配置
```

#### US-EQ-002 配置故障模式
```
作为 知识专家，
我希望 能够配置机床设备的故障模式分类（关联子系统，支持多级分类），
以便 为健康指标提供故障分类基础。

验收标准：
- 故障模式关联**机床型号模板**中的子系统
- 区分关键故障和一般故障
- 每个故障模式有唯一代码和名称
```

### 📐 领域2：规则配置域 (Rule Context)

#### US-RL-001 创建检测规则
```
作为 工艺工程师，
我希望 能够通过公式脚本配置检测规则（Aviator表达式），
以便 灵活定义场景识别和报警触发条件。

验收标准：
- 支持变量引用和逻辑运算
- 支持调用预置检测函数（LIMITS/TREND/DRIFT/DEBOUNCE）
- 规则通过标签分类（场景/报警/故障等）
- 规则可启用/禁用
```

#### US-RL-002 配置健康指标（报警码）
```
作为 工艺工程师，
我希望 能够配置健康指标（报警码），绑定场景规则、报警规则和消除规则，
以便 在特定场景下触发报警，并在条件恢复时自动消除。

验收标准：
- 每个报警码包含：报警内容、关联故障模式、严重等级
- 配置场景规则：定义何时启用该指标
- 配置报警规则：定义触发报警的条件
- 配置消除规则：定义自动消除报警的条件（可选）
- 配置应急措施和关联扣分
- 支持报警/预警两种配置类型
- 当消除规则满足且有未处理报警时，系统自动消除报警
- **支持一键开启/关闭健康指标（可按报警/预警类型分别操作）**
```


#### US-RL-003 使用预置检测函数 【已合并到 US-RL-001】
```
作为 工艺工程师，
我希望 能够在规则公式中调用预置的检测函数，
以便 快速实现常用的检测算法。

验收标准：
- LIMITS：物理限检测
- TREND：趋势检测（连续N点单调变化）
- DRIFT：漂移检测（相对基线偏移）
- DEBOUNCE：防抖动处理

> **该功能已合并到 US-RL-001 创建检测规则（公式脚本详细设计）**
```
#### US-RL-004 机床报警码自动集成
```
作为 工艺工程师，
我希望 系统能够自动接收机床上报的报警码，并自动创建为健康指标，
以便 快速建立机床自身的报警监控，无需人工逐一配置。

验收标准：
- 机床上报报警码时，自动检查是否存在对应的健康指标
- 若不存在，自动创建健康指标（默认来源为ALARM_CODE，默认等级和扣分）
- 自动创建的指标默认启用，即刻参与健康评估和故障诊断
- 自动创建的指标默认关联"未分类故障"模式，支持后续修改归类
- 支持后续在界面上修改自动创建指标的属性（等级、扣分、名称等）
- 支持在统一的健康指标列表中查看和管理（区分规则来源和报警码来源）
```

#### US-RL-005 配置漂移检测模板
```
作为 工艺工程师，
我希望 能够配置漂移检测模板，设置检测场景和参数，
系统根据配置实时检测参数偏离基线的情况，自动触发报警/预警，
以便 提前发现设备劣化趋势，避免故障发生。

验收标准：
【漂移检测模板配置】
- 独立菜单入口：健康指标 → 漂移检测模板
- 设置检测场景名称（如：主轴振动监测、刀具磨损追踪）
- 选择检测指标（设备数据点）
- 配置检测基准（选取历史时间段作为基线）
- 配置报警阈值（均值偏差%）和预警阈值（均值偏差%）
- 支持启用/禁用开关（启用后自动创建对应的健康指标）

【实时漂移检测】
- 基于配置的基线数据，实时计算参数偏差
- 支持均值、最大值、标准差三种偏差检测
- 偏差超预警阈值时创建预警记录
- 偏差超报警阈值时创建报警记录
- 支持配置检测窗口、冷却时间等参数

【与 Run-to-Run 报表联动】
- 从故障诊断记录可打开 R2R 对比分析页面进行验证
- R2R 报表支持通过模板快捷选择，也可手动选择任意参数
```

### 📊 领域3：健康评估域 (Health Context)

#### US-HE-001 计算部件健康评分
```
作为 系统，
当 检测规则触发或定时触发时，
我希望 能够根据报警扣分、预警扣分和寿命因素计算部件健康评分，
以便 量化单个部件的健康状态。

验收标准：
【评分公式】（加权综合评分）
- 部件评分 = (故障健康分 × 权重1) + (寿命健康分 × 权重2)
- 故障健康分 = 100 - 报警扣分 - 预警扣分
- 寿命健康分 = 100 × (1 - 已用时长/设计寿命)，最低为 0
- 默认权重：故障健康分 70%，寿命健康分 30%

【寿命配置】
- 设计使用寿命：配置在设备结构节点中，默认 10 年（87600 小时）
- 已使用时长：通过设备累计在线时间采集
- 重点部件的使用次数/时长：单独定制实现，通过预警规则扣分影响评分

【其他规则】
- 不同报警级别扣分不同
- 评分实时更新
- 报警消除后（自动消除/人工处理），对应扣分恢复
```

#### US-HE-002 汇总设备健康评分
```
作为 车间主管，
我希望 能够查看设备整机的健康评分（由子系统加权汇总），
以便 快速了解设备的整体健康状况。

验收标准：
- 子系统评分 = Σ(部件评分 × 部件权重)
- 整机评分 = Σ(子系统评分 × 子系统权重)
- 评分对应健康/预警/故障三种状态
```

#### US-HE-003 查看设备健康总览
```
作为 车间主管，
我希望 能够在一个看板上查看所有设备的健康状态（绿/黄/红），
以便 快速识别需要关注的问题设备。

验收标准：
- ≥90分：健康（绿色）
- 60-89分：预警（黄色）
- <60分：故障（红色）
- 支持按状态筛选
```

#### US-HE-004 触发健康评估
```
作为 设备管理员，
我希望 健康评估能够在以下时机自动触发：定时、报警发生、手动触发，
以便 及时获取最新的设备健康状态。

验收标准：
- 支持配置定时评估周期
- 报警发生时自动触发相关设备评估
- 支持手动触发单台或批量设备评估
```

### 🔍 领域4：故障诊断域 (Diagnosis Context)

#### US-DG-004 查询报警记录 / 故障诊断
```
作为 维修工程师，
我希望 能够在故障诊断菜单中查看所有设备的故障诊断记录，
以便 快速了解当前需要处理的故障，并选择进行诊断或处理。

验收标准：
【故障诊断列表展示】
- 展示字段：首次报警时间、最近报警时间、设备名称、报警码、报警内容、严重程度、来源
- 使用 Tab 页区分："未处理" 和 "已处理"
- 未处理记录存储在当前表，已处理记录归档到历史表
- 默认按首次报警时间倒序排列
- 支持分页查询

【筛选条件】
- 支持按时间范围、严重程度筛选
- 支持关键词搜索（报警码、设备名称、报警内容）

【记录来源】
- 报警触发：由报警自动创建，关联 IoT 报警记录
- 手工新增：用户手动创建故障诊断记录

【手工新增】
- 选择设备
- 选择报警码（按故障模式分组显示健康指标）
- 严重程度自动填充（来自报警码配置）
- 填写故障描述

【处理方式】（4种）
- 故障诊断：启动诊断树进行故障排查
- 直接处理：填写处理措施后关闭
- 忽略：无需处理，直接关闭
- 自动消除：由系统根据消除规则自动触发（条件恢复正常时）

【批量操作】
- 提供"一键忽略"按钮，批量忽略所有未处理记录

【数据归档】
- 处理完成后，记录从当前表移动到历史表（DELETE + INSERT）
- 自动消除的记录同样归档，handle_type = AUTO_CLEAR

【详情页】
- 展示报警详细信息和数据趋势图（支持多参数曲线）
- 趋势图显示首次报警时刻前后各1分钟的数据变化
```
